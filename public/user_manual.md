# あわい屋ZEROS - 各立場向け使用マニュアル

本システムは、医療・福祉現場を想定したマルチテナント型（複数組織対応）の学習プラットフォームです。
ユーザーの権限（立場）に合わせて最適なメニューと画面が提供されます。

---

## 1. 受講者 (LEARNER) 向けマニュアル
一般スタッフ（看護師、セラピスト、事務員等）が自身の学習を進めるためのマニュアルです。

### 🔑 ログインとトップ画面
1. ブラウザで本番環境のURL（ `https://awaiya-zeros-platform.vercel.app/` ）にアクセスします。
2. 管理者から案内された **「メールアドレス」** と **「パスワード」** を入力してログインします。
3. ログインに成功すると、受講者用の **「ホームダッシュボード」** が表示されます。

### 📊 ダッシュボードの構成
* **学習ステータス（カード）**:
  「学習中のコース」「完了したコース」「全体の累計学習時間」が視覚的に表示されます。
* **期限付き必須研修**:
  組織管理者から「受講期限」が設定された最重要の研修が最上部に並びます。期限までに必ず受講を完了させてください。
* **最近の学習**:
  前回途中まで進めた研修に、1クリックで戻って学習を再開できます。

### 📖 研修の進め方（レッスンの受講）
1. 左側メニューの **「研修一覧」** から、学びたいカテゴリ（例：「医療安全・感染対策」）やコースを選択します。
2. コース詳細画面には、受講すべき「レッスン」が並んでいます。
   * 🎥 **動画レッスン**: 再生ボタンを押して動画（YouTube等）を視聴します。
   * 📄 **PDF / 各種資料**: 画面に表示されるリンクからPDFを閲覧します。
   * 📝 **記事（解説テキスト）**: 画面上でシステムオリジナルの解説文（手順等）を直接読みます。
3. 各レッスンを最後まで進め、ページ下部の **「受講完了（または次のレッスンへ）」** ボタンをクリックすることで、進捗率（％）が自動的にカウントされます。
4. コース内のすべてのレッスンが「受講完了」になると、コース全体のステータスが **「修了（Completed）」** に変わります。

### 🗺️ ロードマップ（学習パス）の受講
* 左メニューの **「ロードマップ」** には、病院や部署が定めた「新卒向けロードマップ」などが並んでいます。
* 「Step 1 ➔ Step 2 ➔ Step 3」と、ステップに沿ってコースが体系的に並んでおり、順番通りに受講を進めることができます。

## 2. 組織管理者 (ORG_ADMIN) 向けマニュアル
各組織（例：「あわい屋総合病院」の教育担当者、看護部長など）が自組織の研修運用やスタッフの受講状況を管理するためのマニュアルです。

### 👥 受講者（スタッフ）の管理
1. 左メニュー of **「ユーザー管理」** をクリックします。
2. 自組織に所属するスタッフの一覧が表示されます。
3. **「ユーザー招待 / 新規登録」** ボタンをクリックします。
4. スタッフの「氏名」「メールアドレス」「所属部署（看護部、リハビリ科等）」「役職」を入力し、初期パスワードを設定して保存します。
   * ※アカウントの一時停止（休職・退職対応）も、一覧のステータス切り替えから簡単に行えます。

### 🎨 組織専用コースの作成・編集
1. 左メニューの **「研修管理」** をクリックします。
2. 画面右上の **「新規コース作成」** ボタンをクリックします。
3. タイトル、説明文を入力し、適切なカテゴリを設定して保存します。
4. 作成したコースの編集画面から、**「レッスンの追加」** を行います。
   * **動画**: YouTubeなどの共有URLを登録します。
   * **PDF**: 参照先のURLを入力します。
   * **記事**: マークダウン形式で、オリジナルの学習用テキストを入力・保存できます。

### 📅 必須研修の設定（アサイン）
特定の研修を、部署単位または組織全体に対して「期限付きの必須研修」に設定します。
1. 左メニューの **「必須研修設定」** をクリックします。
2. **「必須アサインの追加」** をクリックします。
3. 対象となる「コース」、対象の「部署（空欄の場合は組織全体）」、そして「受講期限（日付）」を設定して保存します。
4. 対象スタッフのダッシュボードに「必須研修」として即座にピン留めされます。

### 📈 進捗状況のモニタリング
1. 左メニューの **「受講状況モニタリング」** をクリックします。
2. スタッフごとの「総コース完了数」「現在進行中のコース数」「個別のコースの進捗率（％）」がリアルタイムに表示されます。
3. 部署や氏名で絞り込み、受講が遅れているスタッフへの個別フォローに役立てることができます。

## 3. システム管理者 (SYSTEM_ADMIN) 向けマニュアル
あわい屋ZEROSのシステム本部の担当者が、プラットフォーム全体の運営を行うためのマニュアルです。

### 🏥 組織（テナント）の登録・管理
本システムを導入する新しい病院や事業所を登録します。
1. 管理メニューの **「組織管理」** を開きます。
2. **「新規組織の追加」** をクリックします。
3. 組織の名称（例：「あわい屋リハビリセンター」）を入力し、ステータスを「アクティブ」で登録します。
4. 必要に応じて、その組織が利用する **「部署（Department）」**（例：理学療法部、作業療法部など）の名称を登録します。

### 🌐 システム共通（グローバル）研修の登録
すべての導入組織が共通で使用できる「共通基礎マニュアル」などの研修カテゴリ・コースを作成します。
1. 管理メニューの **「グローバル研修管理」** を開きます。
2. **組織IDを「グローバル（なし）」** としてコースを作成することで、すべての組織の「研修一覧」にデフォルトでそのコースが表示されるようになります。

### 🕵️ 監査ログ（セキュリティログ）の確認
セキュリティ・コンプライアンス維持のため、システム内で行われた重要な操作履歴を監視します。
1. 管理メニューの **「監査ログ」** を開きます。
2. 「どの組織の」「誰が」「いつ」「どのような操作（ユーザー登録、コース削除等）」を行ったのかが時系列で記録されています。不正アクセスや誤操作の追跡が可能です。

## 🚀 本番環境で最初の「システム管理者」を作成する方法
アプリをデプロイした直後は、ログインするためのアカウントがデータベース上にまだ存在しません。以下の手順で最初の「システム管理者アカウント」を作成してください。

1. **Supabaseの認証（Auth）でユーザーを作成**:
   * Supabaseのダッシュボードを開き、左メニューの **「Authentication」**（人物アイコン）➔ **「Users」** を開きます。
   * **「Add User」➔「Create User」** を選択します。
   * 最初の管理者の「メールアドレス」と「パスワード」を入力し、保存します。
   * 作成されたユーザーの **「User ID (UUID)」**（例: `abc123xx-xxxx...`）をコピーしておきます。

2. **アプリのデータベース（public.users）にユーザープロファイルを紐づける**:
   * Supabaseの **「SQL Editor」** を開きます。
   * 以下のSQLの `【コピーしたUUID】` の部分を書き換えて実行（Run）します。
     ```sql
     -- 最初のシステム管理者プロファイルの登録
     INSERT INTO public.users (id, organization_id, name, email, role, is_active)
     VALUES (
       '【コピーしたUUID】', 
       '77777777-7777-7777-7777-777777777777', -- デモ用組織（あわい屋総合病院）
       'システム総括管理者', 
       'あなたの登録メールアドレス', 
       'SYSTEM_ADMIN', 
       true
     );
     ```
   * これで、そのメールアドレスとパスワードでシステム管理者としてログインできるようになります。
   * ログイン後は、アプリ内のUI画面（ユーザー管理）から、他の「組織管理者」や「受講者」を簡単に作成できます。
