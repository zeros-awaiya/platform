# あわい屋ZEROS - システム管理者 (SYSTEM_ADMIN) 向け運用マニュアル

システム本部の管理者用アカウントです。全テナント（組織）の登録・監査ログの監視・共通コンテンツ（グローバルコース）の発行・システム初期セットアップを管理します。

---

## 1. システム管理者の権限範囲

システム本部の管理者用アカウントです。全テナント（組織）の登録・監査ログの監視・共通コンテンツ（グローバルコース）の発行・システム初期セットアップを管理します。

## 2. テナント（導入組織）の新規登録

あわい屋ZEROSプラットフォームを利用する新しい企業・病院等の登録を行います。

1.  管理ダッシュボードから左側メニューの**「組織管理」**を開きます。
2.  **「新規組織 of 追加」**ボタンをクリックします。
3.  組織の「名称（例: あわい屋リハビリ病院）」を入力し、保存します。
4.  必要に応じて、その組織で利用する「部門（看護部、リハビリテーション科、事務部等）」も紐付け登録します。

---

## 3. 共通基礎研修（グローバルコンテンツ）の管理

複数の組織に対して共通で提供する教育コース（例：コンプライアンス研修、医療基礎安全対策など）を作成します。

1.  管理者メニューから**「グローバルコース管理」**を開きます。
2.  組織ID（テナント）を「グローバル（なし）」にした状態で、新規コースとレッスンコンテンツを構築します。
3.  これを有効化すると、登録されているすべての導入組織の「研修一覧」にデフォルトで自動配信されます。

---

## 4. セキュリティ監視（監査ログ）

セキュリティとコンプライアンス要件を満たすため、各組織で行われた重要なアクションをログとしてリアルタイムに記録・追跡します。

*   メニューの**「監査ログ」**を選択します。
*   「操作日時」「実行ユーザー」「所属組織」「アクション種類（ユーザー削除、コース追加、ログイン等）」「対象データのID」が時系列で一覧表示されます。

---

## 5. 本番環境での最初の管理者アカウント作成手順

プラットフォームをデプロイした直後、最初にシステムへログインするための「マスターシステム管理者」の登録手順です。

> **※注意※**：安全上のため、Vercelのフロントエンド画面から直接「システム管理者」を自由に追加することはできない仕組みになっています。必ず以下の手順を踏む必要があります。

1.  **Supabase Authへのユーザー登録**
    *   Supabaseダッシュボードを開き、**「Authentication」** ➔ **「Users」** を選択します。
    *   **「Add User」** ➔ **「Create User」**をクリックし、メールアドレスとパスワードを設定して保存します。
    *   作成されたユーザーの<strong>「User ID (UUID)」</strong>をコピーします。
2.  **プロファイルテーブルへの紐付けSQL実行**
    *   Supabaseの<strong>「SQL Editor」</strong>を開き、コピーしたUUIDを埋め込んで以下のSQLを実行（Run）します。

```sql
-- 最初のシステム管理者の登録SQL
INSERT INTO public.users (id, organization_id, name, email, role, is_active)
VALUES (
  '【コピーしたUUID】', 
  '77777777-7777-7777-7777-777777777777', -- デモ用組織（あわい屋総合病院）
  'システム総括管理者', 
  'zeros.js2025@gmail.com', -- あなたの登録メールアドレス
  'SYSTEM_ADMIN', 
  true
);
```

3.  実行が完了すると、そのメールアドレスとパスワードでシステム管理者としてアプリにログイン可能になります。
